関数も使えるよというチュートリアル

### 波括弧を使える場所

以下の２つだけとのこと。

- テキストとしてJSXタグ内で使う
  <h1>{name}'s To Do List</h1>

- 属性として = 記号の直後に使う
  ダブルクォーテーションをするとただの文字列になるらしい。。。  


### 「ダブル波括弧」で JSX 内に CSS やその他のオブジェクトを含める

オブジェクトも渡せると…！
あー{{ }} っていう記号ではなくて、これ { }(Reactの波括弧) + { } (JavaScriptのオブジェクトのリテラル)だったのか。  

> ですので JS オブジェクトを JSX に渡すときには、オブジェクトを別の波括弧のペアでラップして、person={{ name: "Hedy Lamarr", inventions: 5 }} のようにする必要があります。

そういうことか。  

> 次に {{ と }} を見たときには、JSX の波括弧の中に書かれたオブジェクトに過ぎないということを思い出してください！

ですね！特殊記号だと思わないのが大事！  

### 落とし穴


> インラインの style 属性はキャメルケースで書きます。例えば HTML で <ul style="background-color: black"> となっていれば、あなたのコンポーネントでは <ul style={{ backgroundColor: 'black' }}> になります。

あーこれは落とし穴ですわ。  
Reactではキャメルケースが基本ということをおぼえておこう。  

`aria-*`, `data-*` は例外としてね。

### まとめ

- 引用符内に書かれた JSX 属性は文字列として渡される。
- 波括弧を使えば JavaScript のロジックや変数をマークアップ内に含めることができる。
- 波括弧はタグのコンテンツの中で使うか、属性の場合は = の直後で使う。
- {{ と }} は特別な構文ではなく、JSX の波括弧にくっつくように書かれた JavaScript オブジェクトである。
